# 如何查看导入的法律法规数据

## ✅ 数据已成功导入！

根据验证，数据库中已有 **40,108 条法条**。

## 在 Navicat 中查看数据

### 步骤 1: 确认连接

1. 打开 Navicat
2. 确认连接到 `legal_qa` 数据库
3. 如果看不到连接，请参考 `NAVICAT_CONNECTION_GUIDE.md`

### 步骤 2: 查看数据表

1. 在左侧数据库列表中，展开 `legal_qa` 数据库
2. 找到 `legal_articles` 表
3. **重要：右键点击表，选择"刷新"（或按 F5）**

### 步骤 3: 查看数据

**方法一：直接查看表**

- 双击 `legal_articles` 表
- 在数据窗口中查看
- 如果显示为空，点击"刷新"按钮

**方法二：执行 SQL 查询**

1. 点击"查询" → "新建查询"
2. 复制以下 SQL 并执行：

```sql
-- 查看总数据量
SELECT COUNT(*) AS '总法条数' FROM legal_articles;

-- 查看前20条数据
SELECT * FROM legal_articles LIMIT 20;

-- 按类型统计
SELECT law_type, COUNT(*) AS count
FROM legal_articles
GROUP BY law_type
ORDER BY count DESC;
```

3. 或者直接打开 `check_data.sql` 文件执行

### 步骤 4: 如果还是看不到数据

**检查清单：**

- [ ] 确认连接的是 `legal_qa` 数据库（不是其他数据库）
- [ ] 确认查看的是 `legal_articles` 表（不是其他表）
- [ ] 已刷新表数据（F5）
- [ ] MySQL 服务正在运行

**快速验证 SQL：**

```sql
-- 执行这个查询，应该返回 40108
SELECT COUNT(*) FROM legal_articles;
```

## 通过后端 API 查看

如果后端服务正在运行，可以通过 API 查询：

```bash
# 搜索法条
curl http://localhost:8080/api/legal/article/search?keyword=刑法

# 查看所有法条（前10条）
curl http://localhost:8080/api/legal/article/all
```

## 常见问题

**Q: 为什么 Navicat 中显示表是空的？**
A: 可能是缓存问题，请：

1. 右键表 → 刷新（F5）
2. 关闭并重新打开 Navicat
3. 执行 `SELECT COUNT(*) FROM legal_articles;` 验证

**Q: 数据在哪个数据库？**
A: 数据在 `legal_qa` 数据库的 `legal_articles` 表中

**Q: 如何确认数据真的存在？**
A: 运行 `python check_database.py` 或执行 `SELECT COUNT(*) FROM legal_articles;`
